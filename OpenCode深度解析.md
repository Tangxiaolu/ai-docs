# OpenCode：AI 编程工具的开源答案

最近我把主力的 AI 编程工具换成了 OpenCode。它跟 Claude Code 很像,但我选择它有三个硬核理由:

1. 100% 开源,代码透明可控
2. 不绑定任何厂商,自由切换模型
3. 终端党的终极武器

OpenCode 本质上是 Claude Code 的开源替代品,但它解决的是一个更本质的问题——在 AI 编程这个赛道上,我们到底要不要把命运交给一家公司?

## 开源不是情怀,是控制权

很多人觉得开源只是程序员的情怀。但在 AI 工具这件事上,开源的意义远不止于此。

Claude Code 是 Anthropic 官方出品,体验确实很好。但它是闭源的。这意味着你不知道它背后到底做了什么,也无法根据自己的需求去定制。更重要的是,如果有一天 Anthropic 决定调整策略、提价、或者停止维护,你能做的只有接受。

OpenCode 把所有代码都开放出来了。你可以看到它是怎么工作的,可以自己修改,可以贡献代码。这不是技术浪漫主义,这是对工具的真正掌控。

在 AI 时代,工具链的开源尤其重要。因为这些工具已经深入到了你的开发流程里面,成为了你写代码的核心依赖。如果这个依赖是黑盒的,你就永远无法摆脱被动。

## 多模型时代,不要把自己锁死

Claude Code 只能用 Claude 的模型。这在短期内可能不是问题,但从长远来看,这是一个巨大的风险。

AI 模型的进化速度太快了。今天 Claude 可能是最好的,明天 GPT-5 出来了呢?后天 Gemini 又突破了呢?或者你想用本地模型节省成本呢?

OpenCode 的设计哲学就是 provider-agnostic,也就是不绑定任何提供商。你可以用 Claude,可以用 OpenAI,可以用 Google,甚至可以用本地模型。随着模型能力的趋同和价格的下降,这种灵活性会变得越来越重要。

这就像你不会只用一个浏览器,也不会只用一个编辑器。工具应该由你来选,而不是由厂商来限定。OpenCode 把选择权还给了开发者。

## 终端才是效率的归宿

OpenCode 是由 Neovim 用户和 terminal.shop 的创始人开发的。这群人对终端的理解,远超普通开发者。

他们要做的不是把 AI 塞进 IDE,而是把 AI 做成终端原生的体验。内置的 LSP 支持、流畅的 TUI 界面、极致的性能优化——这些都是为终端用户量身定制的。

如果你是终端党,你会懂这种感觉。图形界面的工具再强大,也比不上在终端里行云流水的操作。OpenCode 要做的,就是把 AI 编程这件事,带回到终端的主场。

而且他们明确表示,会持续推进 TUI 的极限。这意味着 OpenCode 不会为了讨好大众而妥协,它就是要服务好那些追求效率的核心用户。

## 客户端/服务器架构的想象空间

OpenCode 采用了客户端/服务器架构。这听起来很技术,但它带来的可能性非常惊人。

这意味着 OpenCode 可以运行在你的电脑上,而你可以从手机、平板、甚至任何设备上远程操作它。TUI 只是其中一个客户端,未来可以有移动端、Web 端、甚至语音端。

想象一下这个场景:你在通勤路上,突然想起一个 bug。掏出手机,连到家里的 OpenCode 服务,直接修复、提交、部署。整个过程不需要打开笔记本,不需要 SSH,因为 AI 就在你的服务器上,随时待命。

这种架构的灵活性,是传统 IDE 插件做不到的。OpenCode 不只是一个工具,它是一个可以不断扩展的平台。

## 写在最后

OpenCode 现在还很年轻,可能没有 Claude Code 那么成熟。但它走的是一条更长远的路。

开源、自由、终端原生、架构灵活——这些特质,决定了它不会被任何一家公司的战略所左右,也不会因为某个模型的兴衰而消失。

AI 编程工具的未来,应该掌握在开发者手中,而不是掌握在某个科技巨头手中。OpenCode 给出了一个答案。
